# Как использовать эту папку

## Назначение

Эта папка — **чистая копия проекта** для загрузки в GitHub, без секретов и локальных файлов.

## Первая загрузка в GitHub

1. **Создай репозиторий на GitHub** (если еще не создан)

2. **Инициализируй Git в этой папке:**
   ```bash
   cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика_github
   git init
   git add .
   git commit -m "Initial commit: ETL pipeline for Room Broom"
   git remote add origin https://github.com/ТВОЙ_ЛОГИН/НАЗВАНИЕ_РЕПО.git
   git push -u origin main
   ```

3. **Проверь:** Открой репозиторий на GitHub → вкладка **Actions** → должен появиться workflow "ETL (ручной запуск)"

## Обновление при изменениях

Когда в основной папке `1_дата-аналитика` появляются изменения:

1. **Обнови эту папку:**
   ```bash
   cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика
   python3 sync_to_github.py
   ```

2. **Закоммить изменения:**
   ```bash
   cd /Users/nikitasmirnov/Desktop/cursor/Работа/1_дата-аналитика_github
   git add .
   git commit -m "Описание изменений"
   git push
   ```

Или просто скажи AI-ассистенту: **"Обнови папку для GitHub"** или **"Закоммить изменения"**

## Что здесь есть

- ✅ Весь код проекта
- ✅ GitHub Actions workflow
- ✅ Документация
- ✅ requirements.txt
- ❌ Без `.env` (секреты)
- ❌ Без локальных файлов

## Важно

- **Никогда не добавляй `.env`** в эту папку
- Всегда проверяй перед коммитом, что секреты не попали
- Используй эту папку только для работы с GitHub
